Color tokens

Every color comes from colors_and_type.css. Click any swatch to copy its token name.

Cyan is scarce

Use brand cyan for a single focal interaction per viewport: primary CTA, active nav pill, or focus treatment.

Navy is structural

Use primary navy for hierarchy and mass-instance controls such as active filter chips, checkboxes, and toggles.

Status is semantic

Success, warning, error, info, pending, and neutral have fixed meanings. Do not repurpose them for decoration.

Charts stay blue

Data visualization uses the viz scale only. Do not use red, orange, green, or amber for chart series.

Brand

Cyan is reserved for one single-instance focal action per viewport.

Primary (navy)

Navy handles hierarchy, structure, and mass-instance controls.

Neutrals

Neutrals carry text, surfaces, borders, dividers, disabled states, and page backgrounds.

Secondary blue

Secondary blue is reserved for data visualization, illustrations, and the deterministic blue avatar.

Status semantics

Status tones have fixed meanings and are separate from data visualization.

Data viz

Charts use the blue-only visualization scale. No red, orange, green, or amber series.

Avatar palette

Six fallback pairs are deterministic; navy is reserved for the logged-in user.

Typography

Inter for UI

All product UI uses Inter through --font-sans. Do not introduce a third typeface.

Mono for data

Use JetBrains Mono for IDs, money, code references, and aligned numeric values.

Sentence case

Use sentence case for body, buttons, and modals. Reserve uppercase treatment for table column headers.

Micro is narrow

--fs-micro is only for table column headers, paired with --ls-wide.

Typeface and letter spacing

Product UI uses Inter through --font-sans. Headings use --ls-tight, which is -0.05em; at a 16px base size that equals -0.8px.

Motion

Subtle by default

Use motion for hover, focus, state changes, and slide-in panels. Avoid decorative animation.

Reduced motion is respected

No behavior inflation

Components — primitives

Buttons

One primary button per viewport. Use danger only for destructive actions and always label icon buttons accessibly.

Inputs

Labels sit above fields. Placeholder text is hint text, never the only label. Error replaces helper text in the same slot.

Badges vs status

Badges have no dot and are for tags or counts. Business status always uses StatusPill.

Avatars and icons

Navy avatar means the logged-in user. Icons inherit color and use Tabler classes, not embedded SVG.

Stack, Grid, and Container enforce token-based gaps and product width constraints.

Components — base

Table default

Default table headers are divider-only. Use this for standard listings such as orders, candidates, and payroll.

Dense exception

Use dense neutral headers only for comparison tables, 6+ columns, horizontal scroll, or stacked similar tables.

Modal restraint

Modals are for deliberate, blocking moments such as destructive confirmations. Do not stack modals.

Tabs

Pill tabs are page-level navigation. Underline tabs are lower-weight sub-tabs inside an existing context.

Select vs combobox

Select is for small finite lists. Combobox is for long lists, search, async lists, or multi-select flows.

Feedback components

Use toast for non-blocking system feedback, tooltip only to supplement labels, and skeleton when the final shape is known.

Components — patterns

Service orders

Use ServiceOrderCard whenever the UI represents an active or completed service order. Flat mode is for dashboard summaries.

Progress and share

Use DonutCard for fraction-first reads and ProportionBar for lightweight inline percentage comparison.

Status consistency

StatusPill is the single status visualization. Use preset business codes and preserve the six tone meanings.

Mass filters

Chip and FilterChip active states are navy, not brand cyan, because they appear in repeated sets.

Audience targeting

AudienceSelector owns the flat and exclusive selection rules. MultiSelectList owns search, all-toggle, and summary behavior.

ChartCard

Use viz tokens only. Line and bar examples show two-series and comparison cases without status colors.

Patterns

Empty by design

Use when the absence is expected, such as no scheduled orders today.

Empty after filtering

Use when data exists but filters remove all results. The next action should be filter recovery.

Empty KPI

Use an em dash plus caption when a metric is missing or not yet measurable. Do not show zero unless zero is real.

Error states

Separate page-level, component-level, and form-level errors so severity matches the surface.

No permission

Tables — dense vs default

Default first

Divider-only headers keep ordinary operational tables calm and scan-friendly.

Dense when scanning breaks

Neutral header fill is allowed for dense comparison work: 6+ columns, similar stacked tables, or horizontal scrolling.

Numeric discipline

Right-align numbers, use tabular mono values, and show em dash for empty values instead of blank or N/A.
